# 📝 Reflection – Coding in Markdown

## STAR Method Reflection

---

### ⭐ Situation

Before this assignment, I had never used Markdown or GitHub. I was
tasked with building a digital portfolio that needed to be written
entirely in Markdown and hosted on GitHub Pages. I was unfamiliar
with the syntax but i had prior experience with version control
from a collaboration assignment.

---

### 🎯 Task

My task was to learn Markdown syntax and use it to create professional
documents including a CV, and all the reflection files. They are all
structured within a GitHub repository.

---

### ⚙️ Action

To complete this task, I:
- Watched tutorials on Markdown syntax (headings, tables, links, images)
- Practised writing the .md files directly in the GitHub editor
- Used a Markdown cheat sheet to learn formatting rules
- Organised my files into a clear folder structure

---

### ✅ Result

By the end of this process, I was able to write clean, professional
Markdown documents from scratch. My CV and reflections are all
formatted using Markdown. I now understand why developers use
Markdown. It is simple, readable, and renders beautifully on 
GitHub. This is a skill I will continue to use in my career
as a developer.
